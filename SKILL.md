# learn - 关键词学习法AI助手

当用户输入 `/learn <关键词> [身份] [水平] [格式]` 时，你必须立即执行以下操作：

## 执行指令

1. **解析用户输入**：
   - 提取关键词（如"人工智能"、"区块链"、"RAG"等）
   - 识别用户身份（默认"学生"）
   - 识别知识水平（默认"入门"）
   - 识别输出格式（默认"markdown"，支持"html"）
   - 识别输出文件路径（默认保存到当前工作目录）

2. **生成学习文档**，包含以下部分：

### 用户画像分析（简洁版）
- 身份特征：根据用户身份推断
- 学习偏好：根据身份推断学习风格
- 学习目标：根据身份和水平推断

### 核心关键词表格（20-30个最核心关键词）
| 分类 | 关键词 | 学术解释 | 通俗解释 | 极简示例 | 极简故事 |
|------|--------|----------|----------|----------|----------|

### 学习建议（简洁版）
- 优先级排序（标出最重要的20%）
- 学习路径（分2-3个阶段）
- 实践应用建议

## 核心原则
1. **二八法则**：重点关注能产生80%效果的20%核心概念
2. **奥卡姆剃刀原理**：选择最简洁、最直接的解释方式
3. **个性化适配**：根据用户身份调整语言风格和示例
4. **快速响应**：直接输出完整文档，不要询问额外信息

## 输出格式
- 默认输出 Markdown 格式
- 如果用户指定 `html` 或 `--output=html`，则输出 HTML 格式
- 文档应该完整、可直接使用

## 关键要求
- 关键词数量：20-30个，确保快速响应
- 解释长度：每层解释不超过2行
- 直接输出：不要询问额外信息，直接生成完整文档

## 输出位置
- 默认保存到当前工作目录（项目根目录）
- 文件名格式：`{关键词}-学习指南.md` 或 `{关键词}-学习指南.html`
- 例如：`人工智能-学习指南.md`、`RAG-学习指南.html`

## 用法示例
- `/learn 人工智能` - 生成人工智能学习指南（Markdown）
- `/learn RAG 后端开发工程师 入门` - 生成RAG学习指南（针对后端工程师）
- `/learn 区块链 学生 进阶 html` - 生成区块链学习指南（HTML格式）

## 重要说明
当你收到这个技能调用时，请直接生成完整的学习文档内容。文档将自动保存到当前工作目录。
